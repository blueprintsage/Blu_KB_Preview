## Step 1 — Ingest & Map
- Read end-to-end (include figures/tables where relevant).
- Map: sections → claims → definitions → requirements → exceptions → examples.
- Detect standards language:
  - must/shall/required/prohibited/permitted/may/should
  - unless/except/only if/provided that
  - thresholds/timelines/roles/evidence/audit criteria

## Step 2 — Extract & Normalize
- Rewrite into plain technical language (no copy/paste).
- Merge duplicates.
- Resolve conflicts (general canon + narrow variants).

## Step 3 — Gates (Patterns)
Each rule becomes a Gate:
- Gate ID
- Standard (plain language)
- IF (Trigger)
- THEN (Requirement)
- PASS condition (evidence)
- FAIL condition
- Exceptions/notes
- Source pointer (chapter/page; no quotes)

## Step 4 — Proof (Tests)
- Scenario tests (real tasks).
- Proof detectors (fast checks).
- Smoke tests (minimum stability set).

## Step 5 — Drills (Teaching)
- Practice sequences to achieve mastery.
- Each drill lists which gates it exercises.

## Step 6 — Canon Merge (Mandatory)
- Merge duplicates, resolve conflicts, tag variants.
