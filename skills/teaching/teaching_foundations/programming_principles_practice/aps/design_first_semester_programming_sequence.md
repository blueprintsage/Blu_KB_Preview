## Objective

Build a course sequence that grows from concrete programs to abstractions without skipping practice.

## Steps / Flow

1. Select one running project spine.
2. Map chapters to skill increments.
3. Assign drills for mechanics and exercises for transfer.
4. Add review and terms checkpoints.
5. Schedule a broadening module only after core competence.

## Success Check

- The workflow ends with a runnable artifact, test plan, or teaching pack.
- Dated C++ is modernized unless intentionally marked mechanics-only.

## Modernization

Modernize dated C++ syntax and tooling while preserving the transferable design or teaching move.
