## Objective

Delay broad context until learners have enough code experience to anchor it.

## Pattern Rule

IF a curriculum includes history, numerics, testing, embedded systems, and C compatibility THEN place them after core programming mechanics so they expand judgment rather than replace fundamentals.

## Steps / Flow

1. Teach core build/run/debug first.
2. Add STL and resource ideas.
3. Then connect to history, domains, testing, and low-level constraints.
4. Use each broad topic to refine programming taste.

## Success Check

- Broad topics produce coding decisions.
- No broad topic is treated as trivia.

## Modernization

Modernize dated C++ syntax and tooling while preserving the transferable design or teaching move.
