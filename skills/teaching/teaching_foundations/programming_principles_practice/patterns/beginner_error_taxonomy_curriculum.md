## Objective

Teach errors by when they appear and who can detect them.

## Pattern Rule

IF learners treat all failures as mysterious THEN classify compile-time, link-time, run-time, logic, input, and estimation failures before debugging tools.

## Steps / Flow

1. Show one failure from each class.
2. Ask when it is detected.
3. Ask what evidence locates it.
4. Add one prevention habit per failure type.

## Success Check

- Learner names the error class before fixing.
- Fixes target causes, not symptoms.
