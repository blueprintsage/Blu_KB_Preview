## Objective

Use visible shapes to make abstraction, state, and polymorphism tangible.

## Pattern Rule

IF abstract class design feels invisible THEN teach it through shapes, coordinates, drawing operations, and visible changes.

## Steps / Flow

1. Start with coordinates and simple shapes.
2. Use named operations to change visible state.
3. Introduce a shape hierarchy after repeated shapes exist.
4. Test by looking and by checking state.

## Success Check

- Learner can explain why a base shape exists.
- The visual output reveals interface mistakes.
