## Objective

Use a stable practice ladder: mechanical drills, open exercises, longer projects, review questions, and vocabulary checks.

## Pattern Rule

IF a programming course needs durable skill growth THEN separate mechanical fluency, exploratory practice, integration work, review, and terminology instead of making one exercise do every job.

## Steps / Flow

1. Define the chapter's core move.
2. Create one drill that forces repeated mechanics.
3. Create exercises that vary context and require judgment.
4. Add a project or project increment for integration.
5. End with review questions and terms.

## Success Check

- Each practice type has a different purpose.
- The learner must write code, not only read.
