## Objective

Use multiple implementations of the same task to teach abstraction and hidden complexity.

## Steps / Flow

1. State the shared task.
2. Show low-level implementation.
3. Show higher-level implementation.
4. Show scripting/high-level implementation.
5. Ask what each version hides or exposes.
6. Tie comparison to design choice, not superiority.
