## Objective
Move a novice from a word problem to code through a small design artifact.

## Steps / Flow
1. Restate the problem in plain language.
2. List inputs, outputs, and constraints.
3. Write pseudocode using sequence, selection, and repetition words.
4. Translate one pseudocode line at a time into C++.
5. Test with one normal case, one boundary case, and one invalid case.

## Notes
The source emphasizes pseudocode and program design before implementation. This object preserves the method as a workflow.
