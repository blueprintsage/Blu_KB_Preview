## Objective
Use small broken programs to teach diagnosis, not guessing.

## Steps / Flow
1. Give a compact program with one or two intentional faults.
2. Require the learner to classify each fault as syntax, runtime, or logic.
3. Ask for the smallest correction.
4. Run after correction.
5. Write the general rule that would prevent the error next time.

## Notes
Gaddis uses recurring error-finding exercises. The skill object generalizes that format into a reusable debugging drill.
