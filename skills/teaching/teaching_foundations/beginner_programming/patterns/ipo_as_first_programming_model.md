## Objective
Teach novices to see every small program as inputs transformed by processing into outputs.

## Steps / Flow
1. Name the information the program must receive.
2. Name the calculation or decision the program performs.
3. Name the visible result the program must produce.
4. Turn the three-part model into pseudocode before writing C++.
5. Verify the finished program against the same input, processing, and output contract.

## Notes
Gaddis repeatedly frames beginner programs around input, processing, and output. The reusable skill is to turn that framing into a simple design checkpoint.
