## Objective

Turn course examples into repeatable checks instead of one-off terminal runs.

## Steps / Flow

1. Choose tiny deterministic inputs.
2. Compute expected outputs.
3. Run the program automatically.
4. Compare with tolerance for floating point.
5. Report pass/fail clearly.
6. Keep tests with the example.
