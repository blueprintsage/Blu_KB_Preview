## Objective
Debug small programs by tracking variables and branches before reaching for tooling.

## Steps / Flow
1. Choose the smallest input that reproduces the confusion.
2. Make a table of variables and output.
3. Advance one statement at a time.
4. Record branch decisions explicitly.
5. Compare the trace to expected behavior and fix the first mismatch.

## Notes
The source introduces hand tracing as a beginner debugging method. The reusable pattern is state visibility before tool dependence.

Modernized into current C++ teaching practice without copying legacy source wording.
