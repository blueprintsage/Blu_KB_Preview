## Objective
Practice choosing the loop form that matches the control condition.

## Steps / Flow
1. For each scenario, mark whether the repeat count is known, unknown, or at least once.
2. Choose for, while, or do-while from that control fact.
3. Write the loop header before the body.
4. Add the update or read step.
5. Check for off-by-one and infinite-loop risks.

## Notes
Gaddis explicitly teaches loop selection as a software-engineering decision. PASS stores it as a repeatable drill.
