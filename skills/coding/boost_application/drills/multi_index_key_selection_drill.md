# Multi Index Key Selection Drill

## Purpose

Select container indexes from query requirements instead of adding maps reactively.

## Use When

A learner needs hands-on practice applying the corresponding Boost/modern C++ decision.

## Pattern

1. List all lookup queries for one record type.
2. Group equivalent queries.
3. Choose primary and secondary indexes.
4. State mutation consequences.
5. Reject any index without a real query.

## Modern C++ Note

Use modern standard facilities where they express the same idea more directly; keep Boost forms when they teach portability or remain the strongest tool.
