# Boost Or Standard Choice Drill

## Purpose

Decide whether a Boost component should remain, be wrapped, or migrate to a standard library equivalent.

## Use When

A learner needs hands-on practice applying the corresponding Boost/modern C++ decision.

## Pattern

1. Pick one Boost type in a legacy file.
2. Name the role it serves.
3. List the standard replacement, if any.
4. Check semantic differences and compiler baseline.
5. Choose keep, wrap, or replace.

## Modern C++ Note

Use modern standard facilities where they express the same idea more directly; keep Boost forms when they teach portability or remain the strongest tool.
