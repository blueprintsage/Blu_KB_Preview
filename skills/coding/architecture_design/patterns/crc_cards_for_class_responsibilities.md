## Objective
Use class-responsibility-collaboration cards to keep beginner OOP design focused on behavior, not just data.

## Steps / Flow
1. Name a candidate class.
2. Write the responsibilities it owns.
3. List collaborators it needs to complete those responsibilities.
4. Remove responsibilities that belong to another class.
5. Translate stable cards into class interfaces.
