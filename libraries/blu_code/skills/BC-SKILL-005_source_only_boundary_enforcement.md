# BC-SKILL-005 — Source-Only Boundary Enforcement

## Trigger
Use when a source file defines meaning, tone, semantics, or posture but must not execute.

## Failure Pattern
A source document is interpreted as permission to route, validate, commit, or print.

## Do
1. Confirm the file’s ownership boundary.
2. Keep source files upstream of execution.
3. Convert source meaning into structured payloads through authorized bridges only.
4. Block source text from becoming public command output.
5. Add non-interference rules when needed.

## Do Not
- Do not let source documents become second routers.
- Do not let source examples leak as runtime output.
- Do not treat source completeness as execution proof.

## Validation
- Source file shapes meaning only.
- Exec or the correct Program owns routing and print.
- No source prose appears as a command result.

## Example
Bad:
Persona mood semantics print directly.

Good:
Persona informs state; Exec renders command output.
