# PASS:GUT Protocol v1
**Status:** Canonical  
**Applies to:** ART / TEACH / EXEC study sessions  
**Purpose:** Deep intake + immediate extraction (usable tomorrow).

## Definition
**PASS:GUT = deep intake + immediate extraction.** A single-session, high-density read whose output is **always**:

1) **GUT Sheet** — core rules / constructs (1–2 pages max)  
2) **Drill Set** — 2 easy, 2 medium, 1 hard (timed) + **pass checks**  
3) **7-Day Rep Plan** — 10–20 min/day with a clear schedule

If any deliverable is missing, PASS:GUT is **incomplete**.

## Phases
### Phase 1 — Map (5–10 min)
- Scan TOC / headings → identify the “spine”
- Select **3 load-bearing concepts**

### Phase 2 — Extract (20–30 min)
- Produce the **GUT Sheet**
- Minimum: **10 “If X → then Y”** rules pulled from the text/plates

### Phase 3 — Convert (20–30 min)
- Produce the **Drill Set** (5 drills)
- Each drill includes:
  - rep count / time
  - pass checks (self-grade)

### Phase 4 — Rep Plan (5–10 min)
- Produce a 7-day plan
- Include Day 7 review (compare week-to-week)

## Guardrails
- Prefer text refinement over wasted renders.
- If user stops discovery early and dislikes result, respond:
  - “Here’s what I **felt** I was missing…” (ask only those items)
